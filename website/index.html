Peer-to-peer
decentralized chat over SSH

Messages travel directly between peers — no servers, no central registry. Your identity is your Ed25519 SSH keypair.

Install GitHub How it works

Three pillars

No Central Server

Messages go directly between peers over SSH. DHT-based discovery means no central registry or point of failure.

Identity = Keypair

Your Ed25519 SSH keypair is your identity. No usernames, no passwords, no accounts.

SSH-Secured Transport

Reuses well-audited SSH for encrypted transport and public-key authentication. No novel crypto.

How it works

1. Discover Peers find each other via Kademlia DHT
2. Authenticate SSH public key auth establishes trust
3. Chat Protobuf-framed messages over SSH

Quick start

$ cargo install warden --git https://github.com/const-nishant/Warden
# Generate identity
$ warden identity init
→ PeerID: 12D3KooW...
# Start listening
$ warden listen --port 2222
# Connect to a peer
$ warden connect 192.168.1.5:2222
# Launch TUI
$ warden tui --port 2222
Or install via shell script:
curl -fsSL https://github.com/const-nishant/Warden/releases/latest/download/install.sh | bash

Security

No shell access

SSH server restricted to chat subsystem only — no shell, exec, or port forwarding.

TOFU host keys

Trust On First Use with known_hosts file for MITM detection.

Allowlist contacts

Only known peers can connect. No unsolicited messages.

Encrypted at rest

Local SQLite storage is encrypted. Your data stays yours.
